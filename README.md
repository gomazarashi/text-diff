# テキスト差分チェッカー

静的なテキスト差分チェックツールです。旧テキストと新テキストをブラウザ内で比較します。

公開URL:
https://gomazarashi.github.io/text-diff/

## 機能

- 行単位の差分表示
- 行単位差分の一列表示・左右比較表示
- 単語単位の差分表示
- 追加・削除・変更なしの色分け
- 長文の行単位差分では未変更部分を折りたたんで表示
- 入力内容の外部送信なし
- 入力内容の保存なし

## ローカル確認

任意の静的サーバーで確認できます。

```bash
python -m http.server 8000
```

ブラウザで以下を開きます。

```text
http://localhost:8000/
```

## 差分処理の回帰・負荷検証

差分処理の回帰確認用に、手動実行の検証ページを用意しています。差分計算は Web Worker で実行されるため、検証中もページが操作不能にならないことを確認してください。

```bash
python -m http.server 8000
```

ブラウザで以下を開き、`検証を実行` を押します。既定では標準検証を実行します。

```text
http://localhost:8000/tests/long-text-regression.html
```

標準検証では、実ページを `iframe` で読み込み、次のケースを行単位比較で実行します。

- 小さな差分
- 1MB 程度の同一テキスト
- 1MB 程度のテキストで数行だけ差分
- 50,000 行の同一テキスト
- 改行の少ない 1MB 程度の長い 1 行

重い中止ケースを確認する場合は、ストレス検証を実行します。

```text
http://localhost:8000/tests/long-text-regression.html?suite=stress
```

ストレス検証では、次のケースを実行します。

- 大きく異なる 100,000 行のテキストで、差分計算が中止されること

全ケースをまとめて確認する場合は、`suite=all` を指定します。

```text
http://localhost:8000/tests/long-text-regression.html?suite=all
```

`PASS` は期待どおり完了または中止したことを表します。`SLOW` は期待どおりの結果ですが、処理に 10 秒以上かかったことを表します。`FAIL` になった場合、またはブラウザタブが操作不能になった場合は回帰として扱います。

ヘッドレスブラウザ等から自動実行したい場合は、URL に `?autorun=1` を付けるとページ読み込み後に検証が開始されます。

## 注意

このツールは入力されたテキストをブラウザ内で処理します。入力内容はサーバーへ送信・保存されません。ただし、機密情報・個人情報・パスワード等の入力は避けてください。
